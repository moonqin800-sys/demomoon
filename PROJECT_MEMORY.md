## 3. 固定协作规则

1. 每次更新管理后台原型前，先执行 `git pull` 拉取 `demomoon` 最新代码。
2. 每次更新移动端原型前，先执行 `git pull` 拉取 `mobilemoon` 最新代码。
3. 每次生成新的原型页面后，需要询问是否上传到 Git。
4. 后续新增功能基于当前最新原型继续迭代。
5. 保留历史版本文件，新需求生成新的版本号文件，并同步更新入口文件。
6. 只提交本次需求相关文件，不把无关 PPT、视频、临时文件夹一起提交。

## 5. 已完成的重要原型改动

礼品库：

- “新增礼品”改为“同步礼品”，强调礼品数据来自上游系统同步，不是后台手工新增。
- 去掉重置按钮。
- 去掉顶部统计板块：礼品总数、启用礼品数、待补素材、活动绑定。
- 列表去掉“复制”操作。
- 礼品库存 / 礼品库基础信息中，礼品 code、礼品名称不可修改。

活动礼品配置：

- 去掉“批量停用”。
- “添加礼品”改为“同步数据”。
- 同步数据前强调需要先选择活动。
- 同步数据弹框改为轻提示流程：同步中、同步完成，不展示具体添加绑定内容。
- 列表去掉排序、展示时间、状态字段。
- “覆盖配置”文案最终调整为“素材自定义”。
- 活动素材自定义支持修改产品别名。

会员服务轮播图管理：

- 左侧导航调整为“礼品中心”一级分组，下挂“礼品库 / 活动礼品配置 / 操作记录”三个二级菜单。
- 新增独立一级菜单“会员服务轮播图管理”。
- 支持礼品页轮播 Banner 的新增、编辑、排序、上下架、筛选、发布。
- 支持配置跳转类型：外链 H5、小程序页面、不跳转。
- 支持配置 H5 链接、小程序 AppID、小程序页面路径、生效时间和运营备注。
- 右侧消费者端预览支持展示“我的礼品页”轮播 Banner 效果。
- 页面内展示接口配置 JSON，方便研发评审字段结构。
- 去掉页面内水平导航，页面切换仅保留左侧导航。
- 会员服务轮播图管理页去掉页面位置、导出配置和顶部统计概览区块，页面首屏直接进入筛选、列表与编辑配置。

## 7. 后续迭代建议

后续新增需求优先围绕这几条主线处理：

1. 上游同步：明确同步字段、同步状态、失败提示、差异更新策略。
2. 礼品素材：主图、活动展示图、详情图、物料图的尺寸规范和校验。
3. 活动配置：活动维度只做绑定、启停、展示策略和素材自定义。
4. 权限风控：礼品启停、素材自定义、活动绑定建议记录操作日志。
5. 验收体验：增加消费者端预览，降低运营配置错误。
